## Executive Summary

Dan Shipper, CEO of Every, shares insights from building an AI-native company that runs 4 software products with just 15 people, where 99% of code is written by AI agents (Claude Code, Cursor, etc.). The talk introduces "compounding engineering" - a new paradigm where each feature makes the next feature easier to build, rather than harder. Key insights include: there's a 10x difference between 90% and 100% AI adoption; single engineers can build and maintain complex production apps; and AI enables entirely new organizational structures like demo culture, instant onboarding, and cross-product collaboration without traditional abstractions.

### [Compounding Engineering Framework](https://www.youtube.com/watch?v=MGzymaYBiss&t=494s)
**[08:14](https://www.youtube.com/watch?v=MGzymaYBiss&t=494s)** - AI necessitates entirely new engineering primitives and processes

**[08:51](https://www.youtube.com/watch?v=MGzymaYBiss&t=531s)** - **Definition:** Traditional engineering makes each feature harder to build; compounding engineering makes each feature easier to build

**[09:14](https://www.youtube.com/watch?v=MGzymaYBiss&t=554s)** - **The 4-step loop:**
1. **Plan** - Make detailed plans (critical when working with agents)
2. **Delegate** - Tell the agent to do it
3. **Assess** - Tests, trying it, code review, agent code review
4. **Codify** - The "money step"

**[09:48](https://www.youtube.com/watch?v=MGzymaYBiss&t=588s)** - **Codify step explained:** Take all tacit knowledge from planning, delegation, and assessment and compound it back into prompts

**[10:01](https://www.youtube.com/watch?v=MGzymaYBiss&t=601s)** - Store in claude.md files, sub-agents, slash commands - create a library of accumulated knowledge

**[10:12](https://www.youtube.com/watch?v=MGzymaYBiss&t=612s)** - Turn tacit knowledge into explicit collection of prompts for the entire organization

## Key Insights

1. **The 100% Threshold**: There's a massive difference between 90% and 100% AI adoption - the last 10% enables entirely new workflows

2. **One Developer, One Product**: With AI agents, a single engineer can build and maintain production-grade software that would have required a team

3. **Code as Communication**: The shift from code libraries to prompt libraries - sharing knowledge through instructions rather than abstractions

4. **Demo > Memo**: Lower cost of prototyping enables demo-first culture instead of document-first culture

5. **Fractured Attention Productivity**: Managers and people with fragmented schedules can contribute meaningful code by delegating to agents

6. **Stack Agnostic**: No need to standardize on frameworks/languages - AI handles translation between different tech stacks

7. **Living Documentation**: claude.md files and slash commands become organizational memory that compounds over time
